## 三、測試資料建立步驟

### 方法一：使用 Seed 腳本（推薦）

```bash
# 在專案目錄執行
npx prisma db seed
```

此腳本會建立完整測試資料，包含所有模組的關聯資料。

### 方法二：手動建立

1. **員工**：主檔管理 → 員工管理 → 新增員工
2. **場地**：主檔管理 → 場地管理 → 新增場地
3. **供應商**：主檔管理 → 供應商管理 → 新增供應商
4. **銀行帳戶**：帳務管理 → 銀行帳戶 → 新增帳戶
5. **場次**：場次管理 → 新增場次
6. **排班**：排班管理 → 選擇場次 → 指派員工
7. **菜單**：菜單管理 → 品項管理 → 新增品項
8. **收付款**：帳務管理 → 收款/付款管理 → 新增記錄

### 方法三：API 測試腳本

```bash
# 確保已安裝 curl
chmod +x scripts/api-test.sh
BASE_URL=https://your-domain.railway.app ./scripts/api-test.sh
```
